# 👋 はじめまして

東京都在住のWebエンジニアです！
受託・自社開発企業の正社員 → 自社開発フリーランス。
直近 1 年はREST API 実装・改修、バリデーション仕様設計、DB 変更提案、SQL による運用データ提供を担当しつつ、フロントエンドのリード（設計判断・レビュー・新人教育）を兼務しています。


## 💼 About

- 経験: 約 2 年半（正社員 → フリーランス）
- 直近の役割: スタートアップ自社サービス 0→1 開発で、フロントエンドリード + バックエンド（API 実装・仕様レビュー・DB 設計）を兼務
- バックエンド実績:
  - Express で REST API のエンドポイント追加・改修（ギフティング・配信・ポイント系）
  - Zod による入力検証・例外処理の実装
  - バリデーション仕様の洗い出し・ドキュメント化を主導（CTO レビュー経て確定）
  - DB スキーマの課題発見・変更提案（1:N 化・中間テーブル追加等）
  - SQL による運用データ抽出・集計 → 非開発部門への定期レポート
  - CTO 策定の API 仕様に対し、リクエスト/レスポンス構造の過不足をレビュー・改善提案
- フロントエンド実績: 20 画面超の SPA を FSD アーキテクチャで設計・実装（リリース後 2 ヶ月で累計会員登録 1,800 名超）
- AI 活用: Claude Code / Cursor / Codex 等を業務導入し、チーム運用ルールの設計・整備まで担当

## 🔨 実績ハイライト

- REST API の設計・実装・改修を 2 プロジェクトで継続的に担当（Express / Zod / PostgreSQL）
- バリデーション仕様を DB 制約と実用性の両面から設計・ドキュメント化し、FE/BE 間の認識齟齬を防止
- 仕様未確定の 0→1 開発で、画面遷移図・状態遷移表・例外パターンを先行作成し、大規模な手戻りなくリリース
- 4 社横断の大規模物流システムでフロントエンドリードとして期限内リリースを完遂
- GitHub Actions による CI/CD 構築、PR レビュー体制の整備

## 🚀 キャリア方向性

フロントエンドの設計・実装経験を土台に、バックエンドエンジニアとしての専門性確立を目指しています。
次のステップとして、認証認可・例外設計・テスト戦略・ログ/監視といった基盤技術を実務で本格的に担当できるバックエンドポジションを志向しています。
